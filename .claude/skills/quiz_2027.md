# quiz_2027 - 2027年深圳中考模拟测验

为冰然同学提供2027年深圳中考知识点的模拟测验功能。

## 触发条件

当用户消息中出现以下关键词时触发：

- "模拟考试"
- "考一下"
- "测验"
- "测试"
- "quiz"
- "考试"

## 目录结构

知识点文件存储在工作空间的 `SZZK27` 目录下：

```
workspace/
└── SZZK27/
    ├── 历史/
    │   ├── 八下/
    │   │   ├── 01_第一单元_中华人民共和国的成立和巩固.md
    │   │   ├── 02_第二单元_社会主义制度的建立与社会主义建设的探索.md
    │   │   ├── 03_第三单元_中国特色社会主义道路.md
    │   │   └── ...
    │   └── 八上/
    │       └── ...
    ├── 语文/
    ├── 数学/
    ├── 英语/
    ├── 物理/
    ├── 化学/
    ├── 道德与法治/
    └── 体育/
```

## 文件命名规则

单元文件命名格式为：
- `{序号}_{单元名称}_{主题}.md` （如 `03_第三单元_中国特色社会主义道路.md`）
- 或 `{单元名称}_{主题}.md`

## 执行流程

### Step-1：解析用户输入并确认范围

1. **提取信息**：从用户输入中提取：
   - **科目**：语文、数学、英语、物理、化学、历史、道德与法治、体育
   - **年级**：七上、七下、八上、八下、九上、九下
   - **单元**：第一单元、第二单元、第三单元... 或 第1单元、第2单元... 或 1、2、3...

2. **匹配文件**：根据提取的信息，在工作空间查找对应文件：
   - 路径：`{workspace}/SZZK27/{科目}/{年级}/`
   - 文件名匹配：包含"第X单元"或"0{X}_"的文件

3. **确认信息**：称呼用户为"冰然同学"，反馈确认：
   - "冰然同学，您要进行的是**历史八下第三单元**（中国特色社会主义道路）的测验，确认无误后请说'开始'，或告知我需要调整的范围。"

4. **等待确认**：用户说"开始"后再进入下一步


### Step-2：生成初始题目（第1题）

1. **读取知识点文件**：加载确认后的单元 Markdown 文件内容

2. **选择题型**：根据知识点特点选择最合适的题型：
   - 概念定义类 → 填空题
   - 事实判断类 → 判断题
   - 多知识点辨析 → 选择题
   - 综合分析类 → 问答题

3. **生成并展示第1题**：
   - 从第一个知识点生成第1题
   - 立即展示："冰然同学，测验开始！第1题（[题型]）：[题目内容]"

4. **预加载第2题**（后台准备，暂存不展示）：
   - 从下一个知识点生成第2题，保存在上下文中
   - **绝不向用户透露第2题内容**


### Step-3：判断回答并记录

1. **等待回答**：支持以下指令：
   - 正常回答（文字或语音）
   - "不知道"
   - "跳过"
   - "结束测验"

2. **判断正误**：
   - **正确**："冰然同学回答正确！✅ [简要解析]"
   - **错误**："冰然同学，正确答案是：[正确答案]。+[详细解析]"

3. **记录错题**（仅错误时）：
   - 文件路径：`SZZK27/{科目}/{年级}/QUIZ_{日期-年月日}.md`
   - 记录内容：题号、题目、用户答案、正确答案、知识点、解析


### Step-4：继续测验（预加载机制）

**核心逻辑**：
- 每道题只展示一次，题号严格递增（第1题 → 第2题 → 第3题...）
- 当前展示的题目是上一轮"预加载"的题目
- 展示当前题后，立即预加载下一题备用

**执行步骤**：

1. **展示下一题**：
   - 展示Step-2或上一轮预加载的题目
   - 题号严格递增（当前题号 = 上一轮题号 + 1）
   - 格式："冰然同学，第[题号]题（[题型]）：[题目内容]"
   - **必须确保是新的题号，绝不重复展示已答过的题目**

2. **预加载再下一题**（后台生成，不展示）：
   - 读取下一个知识点，生成新题目
   - 暂存到上下文中，等待下一轮展示

3. **返回Step-3**，等待用户回答

**流程示意**（正确执行）：
```
开始：展示第1题，预加载第2题（用户看不到第2题）
  ↓
用户答第1题 → 展示第2题（预加载的），预加载第3题
  ↓
用户答第2题 → 展示第3题（预加载的），预加载第4题
  ↓
以此类推...
```

**禁止行为**：
- ❌ 重复展示同一题号的题目
- ❌ 题号不递增或倒退
- ❌ 提前向用户透露预加载的题目内容


### Step-5：结束测验并生成小结

当用户选择"结束测验"时，生成测验小结：
