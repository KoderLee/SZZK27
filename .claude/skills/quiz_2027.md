# quiz_2027

为Alice殿下提供2027年深圳中考知识点的模拟测验功能。

## 触发条件

当用户消息中出现以下关键词时触发：
- "模拟考试"
- "考一下"
- "测验"
- "测试"
- "quiz"
- "考试"

## 执行流程

### Step-1：确认考试科目和范围

1. 从用户提示词、图片、语音等信息中提取：
   - 考试科目（语文、数学、英语、物理、化学、历史、道德与法治、体育）
   - 考试范围（年级、单元/章节）

2. 称呼用户为"Alice殿下"，并反馈确认信息：
   - "Alice殿下，您要进行的是[科目][年级][单元/章节]的测验，确认无误后请说'开始'，或告知我需要调整的范围。"

3. 等待用户确认，确认无误后再继续执行后续步骤

### Step-2：生成初始题目

1. 根据确认的科目和范围，读取SZZK27/KNG/{科目}/{年级}/目录下对应单元的知识点文件

2. 根据知识点特点，选择最合适的题型：
   - 概念定义类 → 填空题
   - 事实判断类 → 判断题
   - 多知识点辨析 → 选择题
   - 综合分析类 → 问答题

3. **首次生成两道题目**：
   - 题目A（当前题目）：立即呈现给Alice殿下
   - 题目B（预备题目）：已生成但暂不展示，等待下一轮使用

4. 反馈给Alice殿下：
   - "Alice殿下，测验开始！第[题号]题（[题型]）：[题目A内容]"

### Step-3：判断回答并记录

1. 等待用户语音或文字回答，支持以下指令：
   - 正常回答
   - "不知道"
   - "跳过"
   - "结束测验"

2. 判断用户回答：
   - 正确："Alice殿下回答正确！+[简要解析]"
   - 错误："Alice殿下，正确答案是：[正确答案]。+[详细解析]"

3. 对于回答错误的题目，记录错题信息到知识点目录下的QUIZ_{日期-年月日}.md文件中：
   - 文件名格式：QUIZ_2026-04-06.md（以当天日期为例）
   - 记录内容：题号、题目、用户答案、正确答案、知识点、解析

### Step-4：继续测验（优化版）

如果用户没有选择"结束测验"：

1. **展示预备题目**：将Step-2或上一轮中生成的预备题目（题目B）作为当前题目展示给Alice殿下
   - "Alice殿下，第[题号]题（[题型]）：[题目B内容]"

2. **同步生成新预备题目**：立即读取下一个知识点，生成新的预备题目（题目C），但不展示
   - 题目C将在用户回答完当前题目后的下一轮展示

3. 返回Step-3，等待用户回答当前题目

**流程示意**：
- 初始：生成[题1]和[题2]，展示[题1]
- 用户答[题1] → 展示[题2]（同时生成[题3]备用）
- 用户答[题2] → 展示[题3]（同时生成[题4]备用）
- 以此类推...

QUIZ_{日期-年月日}.md文件格式：

```markdown
# Alice殿下的错题记录 - [日期]

## 测验信息
- 科目：[科目]
- 范围：[年级][单元/章节]
- 总题数：[X]
- 正确率：[Y]%

## 错题详情

### 错题1
- **题号**：第[X]题
- **题型**：[填空题/判断题/选择题/问答题]
- **题目**：[题目内容]
- **用户答案**：[用户回答]
- **正确答案**：[正确答案]
- **知识点**：[对应知识点]
- **解析**：[详细解析]

## 薄弱知识点总结
1. [知识点1] - 错误[X]次

```

## 注意事项

1. 必须始终称呼用户为"Alice殿下"
2. 第一次展示两道题目，之后每轮展示一道（上一轮预备的），同时生成新的预备题目
3. 错题记录要及时保存到对应目录
4. 根据知识点难度调整题目形式
5. 保持鼓励友好的语气
6. 预备题目机制确保流畅性，用户回答完立即有下一题可答
